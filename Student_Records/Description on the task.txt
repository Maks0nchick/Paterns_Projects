attr_reader(гетер) и attr_writer(сетер):
---
Геттер — это метод, который возвращает значение атрибута объекта, то есть позволяет получить его.
---
class Person
  def initialize(name)
    @name = name
  end

  # Геттер для атрибута name
  def name
    @name
  end
end
person = Person.new("Alice")
puts person.name  # Вывод: Alice
---
В этом методе name возвращает значение переменной @name и называется геттером. Метод name позволяет получить значение @name, но не изменять его.
---
attr_reader: создаёт геттер для каждого атрибута, позволяя читать значение, но не изменять его напрямую.
---
Сеттер — это метод, который позволяет присвоить значение атрибуту объекта, то есть изменять его.
class Person
  def initialize(name)
    @name = name
  end

  # Сеттер для атрибута name
  def name=(new_name)
    @name = new_name
  end
end

person = Person.new("Alice")
person.name = "Bob"  # Изменение имени
puts person.name     # Вывод: Bob
---
Здесь метод name= (с = на конце) является сеттером и позволяет присвоить новое значение атрибуту @name. Теперь мы можем изменить значение переменной @name через person.name = "Bob".
---
attr_writer: создаёт сеттер для каждого атрибута, позволяя присваивать значение, но не читать его напрямую.
---
initialize: Конструктор инициализирует объект с обязательными полями ФИО и опциональными полями для контактов.

Задание №2
---
В Ruby метод to_s используется для того, чтобы задать строковое представление объекта. Это важно, чтобы объект выводился в человекочитаемом виде, а не просто как указатель на место в памяти.
---
Без метода to_s вывод выглядит так: <Student:0x0000024a37cb3da0>
---
С добавлением метода to_s, вывод будет более информативным: Student: 1, Иванов Иван Иванович
student.rb

class Student
  def to_s
    "Student: #{@id}, #{@last_name} #{@first_name} #{@middle_name}"
  end
end
---
require_relative — это метод, который позволяет загрузить и использовать код из другого файла, расположенного относительно текущего файла. В данном случае, student.rb находится в той же директории, что и main.rb, и именно поэтому используется require_relative.

Задание №3-4
---
Выяснил:
---
1. Когда мы используем attr_reader, attr_writer, или attr_accessor, Ruby автоматически генерирует геттеры и сеттеры для указанных атрибутов, и эти методы не дублируют код:
attr_reader :name
def name
  @name
end

attr_writer :name
def name=(value)
  @name = value
end

attr_accessor :name
def name
  @name
end

def name=(value)
  @name = value
end
---
2. Без использования attr_accessor/attr_reader/attr_writer:
class Student
  def initialize(id, name)
    @id = id
    @name = name
  end

  def id
    @id
  end

  def id=(value)
    @id = value
  end

  def name
    @name
  end

  def name=(value)
    @name = value
  end
end
Тут прописываем геттеры и сеттеры для каждого атрибута, что дублирует код.
---
С помощью attr_accessor конструктор остаётся простым, с минимальным количеством строк кода. Мы явно инициализируем только необходимые атрибуты.
так же можно добавлять или удалять атрибуты в массиве или в конструкторе без необходимости изменять логику геттеров/сеттеров вручную.

attr_reader(гетер) и attr_writer(сетер):
---
Геттер — это метод, который возвращает значение атрибута объекта, то есть позволяет получить его.
---
class Person
  def initialize(name)
    @name = name
  end

  # Геттер для атрибута name
  def name
    @name
  end
end
person = Person.new("Alice")
puts person.name  # Вывод: Alice
---
В этом методе name возвращает значение переменной @name и называется геттером. Метод name позволяет получить значение @name, но не изменять его.
---
attr_reader: создаёт геттер для каждого атрибута, позволяя читать значение, но не изменять его напрямую.
---
Сеттер — это метод, который позволяет присвоить значение атрибуту объекта, то есть изменять его.
class Person
  def initialize(name)
    @name = name
  end

  # Сеттер для атрибута name
  def name=(new_name)
    @name = new_name
  end
end

person = Person.new("Alice")
person.name = "Bob"  # Изменение имени
puts person.name     # Вывод: Bob
---
Здесь метод name= (с = на конце) является сеттером и позволяет присвоить новое значение атрибуту @name. Теперь мы можем изменить значение переменной @name через person.name = "Bob".
---
attr_writer: создаёт сеттер для каждого атрибута, позволяя присваивать значение, но не читать его напрямую.
---
initialize: Конструктор инициализирует объект с обязательными полями ФИО и опциональными полями для контактов.

Выяснил:
---
1. Когда мы используем attr_reader, attr_writer, или attr_accessor, Ruby автоматически генерирует геттеры и сеттеры для указанных атрибутов, и эти методы не дублируют код:
attr_reader :name
def name
  @name
end

attr_writer :name
def name=(value)
  @name = value
end

attr_accessor :name
def name
  @name
end

def name=(value)
  @name = value
end
---
2. Без использования attr_accessor/attr_reader/attr_writer:
class Student
  def initialize(id, name)
    @id = id
    @name = name
  end

  def id
    @id
  end

  def id=(value)
    @id = value
  end

  def name
    @name
  end

  def name=(value)
    @name = value
  end
end
Тут прописываем геттеры и сеттеры для каждого атрибута, что дублирует код.
---
С помощью attr_accessor конструктор остаётся простым, с минимальным количеством строк кода. Мы явно инициализируем только необходимые атрибуты.
так же можно добавлять или удалять атрибуты в массиве или в конструкторе без необходимости изменять логику геттеров/сеттеров вручную.

Задание №6
---
Синтаксический сахар — это приём в программировании, позволяющий сделать код более читаемым и удобным для написания, но не добавляющий новых возможностей или функций.
---
self — это ключевое слово, которое ссылается на текущий объект, внутри которого выполняется код. В зависимости от контекста, self может указывать на различные объекты, но его основная цель — показать, к какому объекту обращается метод в текущий момент.
---
class Person
  attr_accessor :name
  
  def initialize(name)
    self.name = name  # Здесь `self` используется, чтобы вызвать сеттер `name=`
  end
  
  def display_name
    puts "Name: #{self.name}"  # Здесь `self` явно показывает, что вызывается метод `name`
  end
end
---
@ — это обозначение переменной экземпляра. Переменная с @ перед именем принадлежит конкретному экземпляру класса и хранит данные объекта. Значения переменных экземпляра не доступны другим объектам, если они не реализованы через геттеры.
---
class Person
  def initialize(name)
    @name = name  # Здесь `@name` — переменная экземпляра, принадлежащая каждому объекту `Person`
  end
  
  def display_name
    puts "Name: #{@name}"  # Используем `@name` для доступа к значению
  end
end

метод valid_name?, который проверяет, состоит ли строка из букв (как русских, так и латинских), а также пробелов и дефисов (для фамилий или имен с дефисами).

Метод valid_github?:
---
Метод valid_github? проверяет, соответствует ли строка формату ссылки на GitHub. Формат ссылки на GitHub выглядит как https://github.com/username, где username может быть буквенно-цифровым значением или содержать дефисы и подчеркивания.
---
Регулярное выражение \Ahttps:\/\/github\.com\/[a-zA-Z0-9_-]+\z проверяет, что ссылка начинается с https://github.com/ и далее следует имя пользователя.
